## Vision

A rock-solid, `go install`-able Morse decoder with a WAV fixture test suite that catches every timing regression, a clean audio source lifecycle, config persistence so it starts ready for the operator's station, and a session log for reviewing what was heard. The TUI is a first-class binary, not a hidden build artifact.

## Accepted Scope

### 1. WAV Fixture Test Suite
- `testdata/` folder with WAV + expected `.txt` pairs
- `decoder_test.go` runs decoder in `-noui` mode against each fixture, diffs output
- Existing `noisy.wav` (currently untracked in repo root) becomes first fixture
- Catches timing regressions on every `go test ./...`

### 2. cmd/ Build Restructure
- `gmain.go` → `cmd/gmode/main.go`
- `main.go` → `cmd/tmode/main.go` (remove `//go:build nobuild` tag)
- `decoder.go` → `internal/decoder/decoder.go`
- Update Makefile: `go build -o gmode ./cmd/gmode`, `go build -o tmode ./cmd/tmode`
- macOS app: `fyne package -sourceDir cmd/gmode`

### 3. Config File (`internal/config`)
File: `~/.config/mode/config.json`

```json
{
  "device":     "Built-in Microphone",
  "filter":     "bandpass",
  "squelch":    2,
  "bandwidth":  200,
  "noise_gate": 0.3
}
```

**Priority order at startup:**
1. CLI `-file foo.wav` → skip device config entirely (do NOT overwrite)
2. Config device found in PortAudio list → auto-connect
3. Config device NOT found → show picker
4. No config file → show picker

**Saved on change:** device, filter, squelch, bandwidth, noise_gate
**Never saved:** wpm, fwpm, wav file path

**Error handling:**
- Missing file → use defaults, no error
- Malformed JSON → log warning to stderr, use defaults
- Save failure (disk full, permissions) → log warning, continue

**First-run setup:** `os.MkdirAll(~/.config/mode, 0755)` called once at startup, not on every save.

### 4. Session Log
- Written to `~/mode-sessions/YYYY-MM-DD_HH-MM.txt`
- Created with `0600` permissions
- `bufio.Writer` — flushed on every `AddText()` call and on app exit
- `os.MkdirAll` on first write
- Disk-full/permission errors → log warning, continue (never crash)

### 5. `go install`-able Module
- Fix `go.mod`: `module mode` → `module github.com/raff/mode`
- Update all internal import paths
- After: `go install github.com/raff/mode/cmd/tmode@latest` works

### 6. Audio Recording (`-record`)
A `-record output.wav` flag on `tmode` and a Record button on `gmode` that saves the raw incoming audio to a WAV file while decoding proceeds normally.

```
Recording flow:
  AudioReader.Read() chunk
       │
       ├──▶ existing pipeline (filter → decode)
       │
       └──▶ wav.Encoder.Write(chunk)   ← new tap (pre-filter, raw audio)

On exit:   flush + close WAV encoder
On error:  disk full / permission → warn + stop recording, continue decoding
```

Primary use case: capture a live session, then drop the file into `testdata/` as a fixture.

### 7. `-verbose` Debug Flag
Logs one line per tone segment to stderr:

```
[verbose] segment: Sound 45ms → dit (ditTime=48ms)
[verbose] segment: Silence 52ms → mark-space
[verbose] segment: Sound 148ms → dah (ditTime=48ms)
[verbose] char: -. = N
```

Invaluable for tuning filter and timing parameters without modifying source.

## Critical Fixes Detail

### AudioReader Race Condition
Replace ad-hoc mutex/wait with `context.Context` cancellation:

```
IDLE ──Open()──▶ OPENING ──started──▶ RUNNING
                                          │
                             ctx.Cancel() │ or EOF/error
                                          ▼
                                      CLOSING ──done──▶ IDLE
```

`MainLoop` selects on `ctx.Done()` between chunks. `Close()` cancels the context and waits for `MainLoop` to acknowledge — no race.

### MorseDecoder sync.RWMutex
- `Decode()` (frequent reads of ditTime, mSpace, wSpace) → `mu.RLock()`
- `SetWpm()` / `SetFwpm()` (rare writes of multiple fields) → `mu.Lock()`
- Rationale: RWMutex preferred over atomics because ditTime and mSpace are read as a group; atomics risk a torn read across the two fields at human-speed parameter changes.

## Error & Rescue Registry

| Method | Failure | Action | User sees |
|--------|---------|--------|-----------|
| `AudioReader.FromAudioStream` | Device not found | Return error to UI | "Device not available" |
| `AudioReader.FromWaveFile` | File missing/bad | Return error to UI | "Cannot open file" |
| `AudioReader.Read` | Device disconnect | ctx cancel → MainLoop exits | Status: "Input lost" |
| `config.Load` | Malformed JSON | Warn stderr + use defaults | Nothing (transparent) |
| `config.Save` | Not writable | Log warning, continue | Nothing (transparent) |
| Session log write | Disk full | Log warning, continue | Nothing (transparent) |
